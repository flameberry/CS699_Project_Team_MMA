Good morning everyone. Today we are presenting "LegalToday", a web application designed to solve the complexity of legal research using Generative AI. Our goal was to move beyond simple keyword matching and create a system that understands legal context.

Limitations:

Dependent on API quotas; English language only.

We identified three main problems. 1. The sheer volume of data. 2. The failure of keyword search. If a user searches for layman terms, traditional databases fail. 3. The time consumption in reading full PDFs. Our solution covers search, summarization, and connecting users to lawyers.

Here is the high-level data flow. 1. The user query is sent to our Flask backend. 2. We send the text to Google Gemini to get a "Vector Embedding" (a numerical representation of the text). 3. We compare this vector against our SQLite database using Cosine Similarity. 4. The top results are sent back to Gemini to generate a summary, which is returned to the user.

Challenges & Solutions

üõë Challenge: AI Latency

Generating summaries for 10 results sequentially took 15+ seconds.

‚úÖ Solution:

Implemented Python's ThreadPoolExecutor to fire 10 API requests in parallel. Reduced time to < 2s.

‚ö†Ô∏è Challenge: Frontend State

History dropdown was closing before clicks registered due to 'blur' event.

‚úÖ Solution:

Added a setTimeout delay in main.js to allow event propagation.

üìä Challenge: Data Consistency

Scraping disparate legal sources resulted in duplicates and "spurious rows".

‚úÖ Solution:

Engineered a robust Pandas cleaning script to sanitize and normalize dataset structure.

The biggest technical challenge was speed. Calling the AI 10 times in a row was too slow. We solved this by using Multithreading in Python, allowing all 10 summaries to generate at the exact same time. We also faced issues with data cleanliness from scraping, which we solved with custom Pandas scripts.
